How it works

A Deterministic Finite Automaton (DFA) is a mathematical model that reads an input string one character at a time and moves between a fixed set of states based on what it reads. At the end, it either accepts or rejects the input.

For this password validator, each state tracks which requirements have been satisfied so far. Reading a new character may move you to a new state if a new requirement is met.

Q — States
{ q0, q1, q2, q3, q4, q5, q6, qacc, qt }
Σ — Alphabet
All printable characters (uppercase, digit, special, lowercase)
δ — Transition
δ(state, char) → next state
q0 — Start state
Begin here. No requirements satisfied yet.
qacc — Accept
All 3 requirements met + length check.
qt — Trap state
Dead end. No escape. Instant reject.
