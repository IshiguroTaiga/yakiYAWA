How it works

A Deterministic Finite Automaton (DFA) is a mathematical model that reads an input string one character at a time and moves between a fixed set of states based on what it reads. At the end, it either accepts or rejects the input.

This validator uses a minimized 5-state DFA. Each state tracks which category of requirements has been satisfied so far — uppercase seen, or digit/special seen — rather than tracking all combinations individually.

Q — States
{ q0, q1, q2, q3, q4 } — 5 states total
Σ — Alphabet
Uppercase (U), Digit (D), Special (S), Lowercase (L)
δ — Transition
δ(state, char) → next state
q0 — Start state
Begin here. No requirements satisfied yet.
q4 — Accept state
Both requirement groups met. Sink accept — loops on all input.
L — Neutral input
Lowercase self-loops at q0, q1, and q4. At q2 it advances to q3 (uppercase path). At q3 any input reaches q4.
